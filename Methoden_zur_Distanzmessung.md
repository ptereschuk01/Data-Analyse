# Методы измерения расстояний

---

## 1️⃣ Евклидово расстояние (Euclidean Distance)

Формула для двух точек  
\(\mathbf{p} = (p_1, p_2, \dots, p_n)\) и \(\mathbf{q} = (q_1, q_2, \dots, q_n)\):

\[
d_E(\mathbf{p}, \mathbf{q}) = \sqrt{(p_1 - q_1)^2 + (p_2 - q_2)^2 + \dots + (p_n - q_n)^2}
\]

- Это «обычное» расстояние между точками в пространстве  
- Идеально подходит, когда важны **реальные расстояния**  
- Пример: на плоскости (x, y), это прямое расстояние между точками  

---

## 2️⃣ Манхэттенское расстояние (Manhattan / City-Block Distance)

Формула:

\[
d_M(\mathbf{p}, \mathbf{q}) = |p_1 - q_1| + |p_2 - q_2| + \dots + |p_n - q_n|
\]

- Сумма абсолютных разниц по всем координатам  
- «Как пешком по улицам города» — отсюда название Manhattan  
- Не учитывает диагонали, только движение по осям  
- Используется, когда движение ограничено сеткой, например, в логистических задачах  

---

## 3️⃣ Косинусное расстояние (Cosine Distance)

Сначала вычисляем **косинусное сходство**:

\[
\cos(\theta) = \frac{\mathbf{p} \cdot \mathbf{q}}{\|\mathbf{p}\| \, \|\mathbf{q}\|} 
= \frac{\sum_i p_i q_i}{\sqrt{\sum_i p_i^2} \sqrt{\sum_i q_i^2}}
\]

Затем расстояние:

\[
d_C(\mathbf{p}, \mathbf{q}) = 1 - \cos(\theta)
\]

- Показывает **угловое различие между векторами**, а не абсолютное расстояние  
- Важно, когда интересует **направление/форма вектора**, а не длина  
- Часто используется в **Text Mining, NLP, рекомендациях**, где векторы — это TF-IDF или эмбеддинги слов  

---

## ⚡ Сравнение методов

| Метод | Что измеряет | Применение |
|-------|-------------|------------|
| Евклид | Прямое расстояние | Геометрия, KNN, Clustering |
| Манхэттен | Сумма абсолютных разниц по осям | Логистика, городская сетка, KNN |
| Косинус | Угол между векторами | Текст, рекомендации, когда важен «направление» |
